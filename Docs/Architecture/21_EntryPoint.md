# 21. EntryPoint

## Назначение главы

Эта глава разбирает `EntryPoint` как самую верхнюю точку запуска проекта.
Хотя по объёму этот код невелик, по архитектурному смыслу он чрезвычайно важен: именно здесь задаётся начальная ось жизни программы.

## Состав каталога

`Source/EntryPoint/` включает:
- `EntryPoint.asm`
- `Loop.asm`
- `Swap.asm`
- `Include.inc`

### Что это показывает

Даже на уровне стартовой зоны проект сразу различает:
- саму точку входа;
- главный цикл;
- механику swap/переключения;
- агрегирующий include.

## `Include.inc`

### Роль

Собирает entry-point модуль и размещает его в выделенной области памяти.

### Архитектурный смысл

Это означает, что даже стартовая точка проекта мыслится как аккуратно оформленный модуль с фиксированным размером и адресом, а не как случайный кусок кода.

## `EntryPoint.asm`

### Что делает код

Точка входа:
- включает прерывания;
- выполняет `HALT`;
- вызывает запуск `Core`;
- затем вызывает запуск `MainMenu`.

### Почему это сильно

Код стартовой точки не пытается сразу инициализировать весь мир, UI, assets и input сам по себе.
Вместо этого он быстро передаёт управление более специализированным частям системы.

## Роль `Loop.asm` и `Swap.asm`

Даже без глубокого разбора их содержимого уже важен сам факт существования этих файлов рядом с точкой входа.
Это показывает, что проект выделяет:
- основной runtime-loop как самостоятельную тему;
- механику переключения/смены состояния как отдельную тему.

То есть lifecycle проекта начинается не только с “запустить функцию”, а с более общей модели:
- есть начало исполнения;
- есть цикл;
- есть переключение/перекладка состояния.

## Архитектурная роль `EntryPoint`

`EntryPoint` выполняет три базовые функции.

### 1. Стабилизирует старт

У проекта есть одна ясная точка входа.

### 2. Не перегружает начало логикой

В стартовом файле нет хаоса из сотни инициализаций.

### 3. Задаёт ось перехода в модули

Именно отсюда начинается переход к `Core`, а затем к `MainMenu`.

## Диаграмма роли в системе

```mermaid
flowchart LR
    EntryPoint --> Core
    Core --> MainMenu
```

## Практический итог главы

`EntryPoint` в проекте — это дисциплинированная стартовая точка с минимальной, но очень правильной ответственностью. Он не делает всё сам, а запускает модульную ось жизни проекта и тем самым задаёт правильную архитектуру самого первого шага исполнения.
