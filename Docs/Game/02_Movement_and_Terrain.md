# Перемещение и стоимость маршрута

## Назначение системы

Система перемещения определяет, какое расстояние отряд способен преодолеть за полученное мировое время и сколько мировых тиков потребуется для прохождения маршрута.

Скорость принадлежит отряду. Стоимость принадлежит текущему участку маршрута. Общая скорость проигрывания игры не изменяет балансную цену пути.

## Величины расчёта

| Величина | Обозначение | Игровой смысл |
|---|---:|---|
| Прошедшее мировое время | `T` | Количество полученных отрядом мировых тиков |
| Базовая скорость | `V₀` | Единицы движения за один мировой тик |
| Коэффициент героя | `H` | Влияние характеристик, навыков и эффектов героя |
| Стоимость участка | `C` | Цена одного четвертьпиксельного шага |
| Расстояние | `L` | Количество минимальных шагов до цели |

## Базовые формулы

```text
Скорость героя = V₀ × H
Полученные единицы движения = T × V₀ × H
Цена пути = L × C
Пройденные шаги = floor(Полученные единицы движения / C)
Время движения = ceil((L × C) / (V₀ × H))
```

Неизрасходованный остаток единиц движения сохраняется. Целочисленный расчёт не должен терять прогресс, если накопленного значения пока недостаточно для очередного шага.

## Базовая скорость и стоимость шага

`MOVEMENT_DEFAULT_SPEED` задаёт базовое количество единиц движения за один мировой тик. Текущее начальное значение — `56`.

Итоговый коэффициент героя пока принимается равным единице. В дальнейшем он рассчитывается из характеристик, навыков, транспорта, экипировки, нагрузки, бафов и дебафов:

```text
HeroSpeed = MOVEMENT_DEFAULT_SPEED × HeroMovementScale
```

`MOVEMENT_DEFAULT_STEP_COST` задаёт базовую стоимость одного четвертьпиксельного DDA-шага. Текущее начальное значение — `4`.

Эти константы независимы: скорость создаёт единицы движения, а стоимость поверхности расходует их.

Для произвольной поверхности:

```text
Четвертьпиксельных шагов за мировой тик =
    MOVEMENT_DEFAULT_SPEED × HeroMovementScale
    --------------------------------------------
                  SurfaceStepCost
```

```text
Пикселей за мировой тик =
    MOVEMENT_DEFAULT_SPEED × HeroMovementScale
    --------------------------------------------
                SurfaceStepCost × 4
```

## Зависимость от условий маршрута

Неблагоприятные условия не замедляют весь мир. Они увеличивают стоимость шага только для перемещающегося по ним отряда.

| Условия перемещения | Стоимость шага | Скорость относительно базовых условий |
|---|---:|---:|
| Благоприятные | `0,5 × MOVEMENT_DEFAULT_STEP_COST` | 200% |
| Базовые | `MOVEMENT_DEFAULT_STEP_COST` | 100% |
| Затруднённые | `2 × MOVEMENT_DEFAULT_STEP_COST` | 50% |
| Тяжёлые | `2,5 × MOVEMENT_DEFAULT_STEP_COST` | 40% |

Коэффициент может быть дробным в правилах, но храниться как целое количество единиц движения. Например, при `MOVEMENT_DEFAULT_STEP_COST = 4` коэффициент `2,5` представляется стоимостью `10`.

Стоимость читается для текущего участка. После фактического перехода на новый участок оставшаяся часть маршрута оплачивается уже по его стоимости.

## Профиль подвижности

Одинаковые условия не обязаны одинаково влиять на разные отряды. Итоговая стоимость может зависеть от типа отряда, транспорта, навыков, экипировки, груза, погоды и активных эффектов.

Профиль подвижности изменяет стоимость конкретного участка для конкретного исполнителя, не изменяя течение глобального времени.

## Пакет мировых тиков

Несколько мировых тиков могут проигрываться одной cadence-эпохой. Фактически применённый масштаб текущей эпохи обозначается `PlaybackScale`:

```text
Полученные единицы движения =
    PlaybackScale × MOVEMENT_DEFAULT_SPEED × HeroMovementScale
```

Флаг активной фазы сообщает cadence-диапазону о доставке нового пакета «мировых тиков». Только при установленном флаге объект начисляет новый бюджет движения. Флаг сбрасывается после завершения `CadenceStep 4`, а уже начисленный `MovementPending` продолжает распределяться до конца cadence-эпохи.

Cadence распределяет нераспределённые единицы движения между проходами планировщика, но не изменяет итоговое расстояние или стоимость пути в «мировых тиках»:

```text
1/2 → CadenceStep 1, 3, 5, 7 → четыре четверти бюджета
1/4 → CadenceStep 2, 6       → две половины бюджета
1/8 → CadenceStep 4          → весь бюджет
```

## Общая скорость игры

`FGameConfig.PlaybackSpeed` является пользовательской настройкой проигрывания, а не характеристикой героя и не множителем стоимости маршрута. Значение по умолчанию — `1`.

Она может определять количество мировых тиков, упакованных в одну cadence-эпоху:

```text
PlaybackSpeed = 1 → PlaybackScale = 1
PlaybackSpeed = 2 → PlaybackScale = 2
PlaybackSpeed = 4 → PlaybackScale = 4
```

Если оставшаяся длительность действия меньше выбранного масштаба:

```text
PlaybackScale = min(PlaybackSpeed, оставшиеся мировые тики)
```

При увеличении общей скорости календарь, перемещение и остальные временные процессы получают одинаковое количество дополнительных тиков. Поэтому действие проигрывается быстрее в реальном времени, но требует того же числа мировых тиков.

Изменение общей скорости во время действия не меняет оставшуюся игровую стоимость маршрута.

## Сухой остаток текущей реализации

Уже работает:

- `PlaybackScale = min(PlaybackSpeed, AdvanceLeft)`;
- за пакет времени объекту начисляется `MOVEMENT_DEFAULT_SPEED × PlaybackScale` единиц движения;
- начисленные единицы сначала попадают в `MovementPending`, затем долями передаются в `MovementBudget`;
- каждый оплаченный DDA-шаг перемещает объект на четверть пикселя по главной оси и, когда требует ошибка линии, по вторичной;
- цена DDA-шага читается из общей таблицы проходимости по текущему `HextileID` и не хранится в объекте;
- после фактического перехода в другой гекс `HextileID` обновляется, поэтому следующий шаг оплачивается по условиям уже нового участка;
- неизрасходованный `MovementBudget` сохраняется между вызовами;
- игрок и AI используют общий DDA-код и общую таблицу проходимости;
- текущие маршруты ведут от центра одного гекса к центру другого, при этом DDA допускает произвольную конечную точку.

Пока является временным или не реализовано:

- коэффициент `HeroMovementScale` принят равным единице; характеристики, навыки, бафы и дебафы ещё не участвуют в расчёте;
- значения таблицы проходимости тестовые и не являются окончательным балансом;
- распределение бюджета по cadence-проходам реализовано для частот `1/2`, `1/4` и `1/8`, но ещё требует проверки в игре;
- анимация AI обновляется после фактического DDA-смещения, а анимация игрока пока связана с каждым cadence-проходом активного движения; самостоятельная система визуального проигрывания ещё не выделена;
- поворот перед началом движения является бесплатной визуальной подготовкой и не расходует единицы движения;
- расчёт полной цены составного маршрута до начала движения ещё не оформлен как отдельная игровая подсистема.
